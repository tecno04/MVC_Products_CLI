## Spanish
Ésta es una práctica realizada en JAVA, donde se implementa el patrón **MVC**, junto a **Streams** y **Optionals**
Por el momento **NO** tiene persistencia en base de datos, solo en Memoria.

Aplicamos diferentes paquetes para poder modelar el patrón **MVC**, donde se diferencia:
.Model
.View
.Controller
.Utils
.Exceptions
.Service
.Repository
.Interfaces

Ademas, se incorpora la libreria de **Lombok** para gestionar de forma mas sencilla.

Para poder usarlo:

# 1. Clona Repositorio:
```bash
git clone https://github.com/tecno04/MVC_Products_CLI.git
```

# 2. Abri tu ide preferido (**IntelliJ** | **Cursor** | **Vs Code**)

# 3. Ejecuta y pruébalo. Tendrás un menú para poder:
     #1. Agregar Productos
     #2. Listar Productos
     #3. Actualizar un Producto (Total o parcialmente por alguna propiedad)
     #4. Borrar un producto.

## English
This is a practice carried out in JAVA, where the **MVC** pattern is implemented, along with **Streams** and **Optionals**
At the moment **NO** has persistence in the database, only in Memory.

We apply different packages to be able to model the **MVC** pattern, where it differs:
.Model
.View
.Controller
.Utils
.Exceptions
.Service
.Repository
.Interfaces

In addition, the **Lombok** library is incorporated to manage it more easily.

To be able to use it:

#1. Clone Repository:
```bash
git clone https://github.com/tecno04/MVC_Products_CLI.git
```

# 2. Open your preferred ide (**IntelliJ** | **Cursor** | **Vs Code**)

#3. Run and test it. You will have a menu to be able to:

     #1. Add Products
     #2. List Products
     #3. Update a Product (Totally or partially due to some property)
     #4. Delete a product.
